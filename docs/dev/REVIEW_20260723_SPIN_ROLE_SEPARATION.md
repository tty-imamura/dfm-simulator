## 1. 設計 — 結合係数方式(第19次裁定 ChatGPT §11.1 の案B)

`single` 粒子に任意属性 **sFrame / sTherm / sRep**(各 0〜2・既定 1)を追加。
単一変数 spin の三役を、既定値では従来と**厳密等価(bit 等価)**のまま分離可能にする:

| チャネル | 式 | 係数の作用先 |
|---|---|---|
| 空間引きずり(A8/E3) | ω_frame = **sFrame**·s·(R/(R+d))^q | 物質の u(E3)・光線随伴(E8R traceRay)・コア差動分にも乗算 |
| 温度(A9)・放射(E11) | T = ½I·(**sTherm**·s)² | 放射冷却 Λ∝T^p・温度色/ヒストグラム/スペクトル/HUD の全表示 |
| スピン斥力(A10/E5′) | ω_rep = **sRep**·s·(R/(R+d))^q | E5′ の対称化斥力 |

**係数を掛けないもの(物理スピン=角運動量の帳簿)**: E9 接触摩擦(速度→スピン変換)・
E10′ スピン拡散・E6′ 反作用トルク・L 保存の勘定・verify のエネルギー勘定(V10/V10b)。
E11 冷却が減らすのも物理スピンで、radE/radL は実減少分 — T7 恒等式は従来どおり閉じる。

実装の要点:

- **hasSep 高速経路**: 全粒子の係数が 1 の宇宙は乗算自体を素通り(`sep?` 分岐)。
  従来プリセット・セーブは演算列まで不変 = **bit 等価**(QA `sep.equivalence` で機械検証)
- validatePreset(single のみ受理・0〜2 クランプ・数値検査)/ SYSTEM_PROMPT に仕様1行
  (zonal/coreMR の前例に倣う)/ 粒子エディタに **s(引)/s(熱)/s(斥)**(EN: s·drag/s·heat/s·rep)
  行を追加 / cloneSimState(A/B)へ転写 / 光線キャッシュキーに sF を追加
- beta SW キャッシュ: `dfm-beta-v1.30-b1`(新開発サイクル開始)。APP_VERSION は 1.29 のまま
  (package.json と `version.sync` 同期 — 昇格時に一括更新の従来手順)

## 4. 較正で得た新知見(記録)

1. **整列ローターリングの u_φ は渦場の構造を持つ**: リングの外側では順行押し上げ
   (+11%@r=200)、**内側では渦の裏側として逆行**(−17%@r=140)。「ハローを揃えれば全域で
   増強」ではない — ChatGPT 第19次 §8.2「半径依存整列」設計の必要性を実測で裏付け
2. **長時間の外縁比較は環境依存で反転する**: 🕶️(kRep=1・ハローのみ sRep=0)の
   sFrame on/off 比は 3000步 0.91・6000步 0.78 と **on が低い**。第19便(kRep=0 全体)では
   6000步 +6.7% だった — κ_s がハローの**物理スピン**を円盤へ拡散し、円盤自身の kRep 斥力が
   効く経路の有無で長時間挙動が入れ替わる。ゆえに QA は **t=0 決定論検査(渦場)+中期安定
   検査**の設計とし、進化量の主張は探索実験(exp-darkrotor)の領域に留めた(4-20 多seed の
   必要性を再確認)
3. **E11 は pinned 粒子をスキップする**(粒子ループ②の先頭 continue)— pinned 天体は
   η_rad>0 でも放射冷却しない。既存仕様だが文書未記載だったため記録(較正時に発見)
4. **新裁定候補(→台帳 4-50)**: E10′ スピン拡散は「熱伝導」(T6)だが、分離後も物理スピンを
   拡散するため「冷たい(sTherm=0)のに熱伝導でスピンを失う」解釈のねじれが残る。
   E10′ に sTherm を掛けるか(熱伝導=温度の拡散と再解釈)は理論変更なので要裁定
