# 仮定物理シミュレーションアプリ モデルルーティング計画

model-routing スキルに基づく計画書。作成日: 2026-07-12。
本プロジェクトは「開発時」(このリポジトリの成果物を作る作業)と「実行時」
(完成アプリがユーザーの iPhone 上で Claude API を呼ぶ処理)を分けてルーティングする。

## 品質基準(受け入れ条件)

成果物全体が満たすべき条件(検証可能な形):

1. `docs/PHYSICS.md` — ユーザー提示の全15法則が公理(A1〜A10)・帰結(T1〜T6)のどれかに対応付き、
   実装可能な計算式(E1〜E10)と既定パラメータ表を持つ。
2. `index.html` — 単一ファイル・外部依存ゼロで、以下を全て満たす:
   - [ ] iPhone Safari 相当のビューポート(390×844)で横スクロールなしに操作できる
   - [ ] 6つのプリセットを選択でき、各プリセットが起動時にコンソールエラー 0 件
   - [ ] 実行 / 停止 / 再開 / 初めから が動作する
   - [ ] 物理パラメータが実行中いつでも編集でき、即時反映される
   - [ ] パラメータをコメント付きで保存・読込・削除でき、リロード後も残る(localStorage)
   - [ ] AIタブ: APIキーは localStorage のみに保存され、生成されたプリセットが選択肢に追加される
   - [ ] APIキー未設定・不正キー・ネットワーク断で日本語のエラーメッセージが表示され、アプリは壊れない
3. 実行時 LLM — few-shot+JSON抽出+アプリ内検証+リトライ+上位モデルフォールバックの段構え
   (docs/HANDOFF_RUNTIME_LLM.md 参照。構造化出力 output_config は実機で
   `400 grammar too large` となったため 2026-07-13 に撤去)。
4. 機械的検査: 埋め込み JS が `node --check` を通過し、headless Chromium で全プリセットの
   スクリーンショットが取得できる。

### 実行時モデル選定の詳細(③)

- 既定: `claude-haiku-4-5` — 200K文脈、64K出力で十分。出力形式は few-shot+JSON抽出+検証で担保
  (構造化出力は本スキーマではコンパイル上限超過のため不使用)。
- フォールバック: `claude-sonnet-5` + `thinking: {"type":"disabled"}`(JSON整形に思考は不要。
  Sonnet 5 は thinking 省略時にアダプティブ思考が既定オンになるため明示的に無効化する)。
- UI でモデルを選択可能(既定 Haiku)。APIキーは `localStorage` のみに保存し、送信先は
  `https://api.anthropic.com` のみ(ヘッダ `anthropic-dangerous-direct-browser-access: true` で
  ブラウザ直接呼び出し。キーが第三者サーバを経由しない)。
- **プロンプトキャッシュは採用しない(意図的な不採用)**: プリセット生成は散発的な操作
  (5分TTL内の連続呼び出しが期待できない)であり、かつ Haiku 4.5 の最小キャッシュ対象は
  4096トークンで本プロンプト(約3.3K)は届かない。キャッシュ書込プレミアム(1.25×)だけ
  払うことになるため、few-shot を厚くしつつ非キャッシュで送る方が安い。

  - 入力 ≈ 3.6Kトークン(システム+few-shot+スキーマ+ユーザー記述)、出力 ≈ 1.2Kトークン

  - **期待値 ≈ $0.013/回(約2円)**
- **想定月間**: ユーザーが月20回プリセットを生成しても **≈ $0.26/月**。
  (参考: 全件 Sonnet 5 なら $0.058/回、全件 Opus 4.8 なら $0.048/回 — Haiku 段構えが最安)
- **料金確認日: 2026-07-12**(platform.claude.com のモデル一覧で確認:
  Haiku 4.5 = $1/$5、Sonnet 5 = $3/$15(2026-08-31 まで導入価格 $2/$10)、

### 実行指示書: UI修正(フェーズ③、確定済み)

**根因**: `index.html` のタブクリックハンドラは、同一タブ再タップ(=パネルを閉じる)の分岐で

CSS の `height:100%` がバッファを縦に引き伸ばす(390×844 実測で 2.103 倍)。

**差分(2箇所・これ以外の変更禁止)**:
1. タブの閉分岐に `setTimeout(resizeCanvas,50);` を追加(開分岐と同じ遅延)。
2. `window.addEventListener("resize",...)` の直後に
   `if(window.ResizeObserver) new ResizeObserver(resizeCanvas).observe($("#canvasWrap"));` を追加
   — パネル開閉は window resize を発火しないため、`#canvasWrap` の寸法を直接監視する
   (iOS Safari 13.1+ 対応。レイアウト起因の同種バグを恒久的に塞ぐ)。
   `render()` は停止中も毎フレーム実行されるため、リサイズによるキャンバスクリアは無害。

**受け入れ条件**: 品質基準 4・5(検証スクリプトは 開→閉 操作後に
`cv.height == round(canvasWrap.height × min(devicePixelRatio,2))` ±2px を3状態で判定)。
