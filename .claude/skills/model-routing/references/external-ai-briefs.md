# 外部AI向け指示書(ブリーフ)テンプレート

画像・動画・音声・3D・楽曲など、外部の生成AIに委譲する成果物のための指示書の書き方。指示書の品質が生成物の品質を決める。「いい感じの画像を」で終わらせない。

## 原則

- **成果物の用途から逆算する**: ゲームのUIアイコンとキービジュアルでは、同じ「画像」でも解像度・スタイル・余白・ファイル形式の要件が全く違う。用途→技術要件→美術方針の順で確定させる。
- **ツールの方言に翻訳する**: 同じ意図でも、ツールごとに効くプロンプトの書き方が違う。指示書は「意図の正本(ツール非依存)」+「ツール別プロンプト案」の二層で書く。ツールの最新仕様・得意分野は生成前にWeb検索で確認する(生成AIツールの進化は速く、暗記に頼ると古い制約を引きずる)。
- **バリエーションと選定基準をセットで**: 1発で当てようとせず、意図的に振ったバリエーションを複数生成させ、受け入れ条件で選ぶ前提の指示書にする。
- **反復を前提に**: 初回生成後の「直し指示」の語彙(何を固定し、何を変えるか)まで指示書に含めると、ユーザーが外部ツール上で自走できる。

## テンプレート: 画像生成ブリーフ

```markdown
# 画像生成指示書: [アセット名]

## 用途と文脈
[どこで何のために使われるか。閲覧距離、背景色、隣接要素]

## 技術要件
- 解像度/アスペクト比: [値](用途からの根拠)
- ファイル形式: [PNG/JPG/SVG等、透過の要否]
- 点数: [必要枚数、バリエーション数]

## 美術方針
- スタイル: [具体的な様式名。参照作品・作家がある場合は「〜風の傾向」として言語化した特徴で書く]
- 色: [パレット指定。HEX値があれば書く]
- 構図: [被写体位置、余白、視点、ライティング]
- トーン: [感情的な狙い]

## 被写体の確定事項
[キャラクターなら: 髪型・服装・持ち物・ポーズ等、揺れてはいけない要素を列挙。
 シリーズ物なら一貫性維持のための固定記述(キャラクターシート)を作る]

## ネガティブ(避けるもの)
[出てはいけない要素: 文字化けテキスト、余分な指、特定の雰囲気、他作品への類似]

## プロンプト案
### [ツールA]向け
[そのツールの記法に沿った完成プロンプト。パラメータ(--ar 等)込み]
### [ツールB]向け
[同上]

## バリエーション指示
[何を軸に何案振るか。例: 構図3案 × 配色2案]

## 受け入れ条件
- [ ] [検証可能な条件。例: 32px縮小時に判読可能、指定パレット外の色が支配的でない]

## 直し指示の語彙
- 固定したい要素が崩れたら: [再指示の定型文]
- [よくある失敗と修正プロンプトの対]
```

## テンプレート: 動画生成ブリーフ

画像ブリーフに以下を追加:

```markdown
## 時間設計
- 尺: [秒数] / フレームレート
- カット構成: [絵コンテ的に。開始フレーム→動き→終了フレームを1カットずつ]
- カメラ: [固定/パン/ズーム等、カットごとに]

## 動きの確定事項
[何が動き、何が動かないか。速度感。ループの要否]

## 音
[別途音声AIに委譲するか、無音か。委譲するなら音声ブリーフを別に作る]
```

## テンプレート: 音声・楽曲生成ブリーフ

```markdown
## 用途と再生環境
[BGM/SE/ナレーション、ループ要否、再生機器]

## 音楽的指定
- ジャンル/テンポ(BPM)/キー/尺
- 楽器編成、ミックスの質感
- 参照する雰囲気(言語化した特徴で)

## 構成
[イントロ→展開→ループ点、秒数割り]

## 技術要件
[形式、サンプリングレート、ラウドネス目標]
```

## 権利・安全上の注意

- 実在の人物の顔・声の生成、実在アーティスト名の直接指定は避け、特徴を言語化した記述に置き換える。
- 生成物の商用利用可否・権利帰属はツールごとに異なる。商用案件では指示書に「利用規約の確認結果」欄を設け、生成前に確認する。
- 他作品のキャラクター・ロゴの再現を求める指示は書かない。
